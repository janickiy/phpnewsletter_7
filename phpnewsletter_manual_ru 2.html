PHP Newsletter — Руководство пользователя

PHP Newsletter — это инструмент email-маркетинтолога для организации E-mail рассылок с вашего сайта, любой информации.

Ниже приведён мануал, дополненный по структуре исходного приложения из архива: Laravel-архитектура, маршруты подписки/отписки, cron-команды, роли пользователей, логирование, макросы и модули администрирования.

1. Назначение системы

PHP Newsletter подходит для новостных, маркетинговых и сервисных рассылок. Система позволяет развернуть собственную платформу email-маркетинга без зависимости от внешних SaaS-сервисов и без обязательной абонентской платы.

  • отправка массовых email-кампаний;
  • управление базой подписчиков и категориями;
  • работа с HTML- и plain text-письмами;
  • отслеживание открытий и переходов по ссылкам;
  • планирование рассылок по расписанию;
  • размещение всей системы и базы данных на собственном сервере.

2. Архитектура приложения

По содержимому архива приложение построено на Laravel 10 и использует стандартную для Laravel структуру проекта. В зависимостях присутствуют PHPMailer, PhpSpreadsheet, League CSV, Laravel Sanctum и DataTables.

Backend:
PHP 8.2+, Laravel 10
База данных:
MySQL 5.6+
Импорт/экспорт:
CSV, TXT, XLS, XLSX, ODS
Отправка:
SMTP, mail(), sendmail

Ключевые каталоги

  • app/ — контроллеры, модели, DTO, middleware, helper-классы и консольные команды;
  • routes/ — маршруты web, api и console;
  • resources/views/ — шаблоны интерфейса и экраны установки;
  • database/migrations/ — миграции таблиц системы;
  • storage/ — логи, временные файлы, кэш;
  • public/ — публичные ресурсы приложения.

3. Основные возможности

3.1. Отправка писем

  • поддержка SMTP, mail() и sendmail;
  • массовая отправка писем;
  • поддержка вложений;
  • выбор формата письма: HTML или plain text;
  • добавление служебных заголовков, включая Precedence и List-Unsubscribe;
  • поддержка DKIM-подписи для повышения доставляемости.

3.2. Шаблоны и персонализация

  • визуальный редактор шаблонов;
  • персонализация писем с помощью макросов;
  • поддержка различных кодировок;
  • предподготовка email-шаблонов для повторного использования.

3.3. Подписчики и категории

  • ведение базы подписчиков;
  • распределение по категориям;
  • массовые операции: активация, удаление, перемещение;
  • встроенная форма подписки;
  • подтверждение подписки и отписка по персональным ссылкам.

3.4. Импорт и экспорт

  • импорт из TXT, CSV, XLS, XLSX, ODS;
  • экспорт данных в TXT и Excel;
  • экспорт статистики и отчётов.

3.5. Аналитика

  • учёт отправленных и неотправленных писем;
  • отслеживание открытий писем для HTML-рассылок;
  • статистика переходов по ссылкам;
  • просмотр логов и выгрузка отчётов.

3.6. Автоматизация

  • расписание рассылок;
  • очереди и фоновая обработка;
  • запуск консольных задач через cron.

4. Разделы административной панели

По найденным контроллерам система включает следующие административные разделы:

  • Templates — шаблоны писем;
  • Category — категории подписчиков и рассылок;
  • Subscribers — база подписчиков, импорт и экспорт;
  • Macros — макросы для персонализации;
  • Schedule — календарь и планирование рассылок;
  • Log — логи отправки и отчёты;
  • Redirect — статистика переходов по ссылкам;
  • SMTP — настройка SMTP-подключений;
  • Settings — общие настройки системы;
  • Users — пользователи административной панели;
  • Update — модуль обновления;
  • Pages — служебные страницы, phpinfo, форма подписки и список cron-задач.

5. Роли и права доступа

В маршрутах приложения обнаружено разграничение прав минимум на две роли:

  • admin — полный доступ к SMTP, настройкам, пользователям и остальным разделам;
  • moderator — доступ к категориям, подписчикам и макросам.

Контроль доступа реализован через middleware permission.

6. Планировщик и cron

В app/Console/Kernel.php зарегистрированы две команды планировщика:

  • emails:send — выполняется каждую минуту;
  • emails:unsent — выполняется каждые 10 минут.

После этого Laravel будет сам запускать отправку писем и обработку неотправленных сообщений по внутреннему расписанию.

7. Системные требования

  • PHP 8.2 и выше;
  • MySQL 5.6 и выше;
  • Apache 2+ с mod_rewrite или Nginx;
  • PHP-расширения: mbstring, zip, curl, iconv, gd;
  • поддержка функции mail() — если планируется этот способ отправки.

8. Установка

  1. Распакуйте архив в каталог сайта, например /phpnewsletter.
  2. Создайте или настройте файл .env.
  3. Установите права доступа 775 для .env, bootstrap/cache, public и storage.
  4. Откройте в браузере адрес установщика:
    http://your-site/phpnewsletter/install/
  5. Пройдите шаги мастера установки: проверка требований, прав доступа, параметры базы данных, создание администратора и завершение установки.

В проекте присутствуют отдельные экраны установщика: requirements, permissions, database, admin, complete и error.

9. Первый запуск после установки

  1. Войдите в административную панель через страницу /login.
  2. Проверьте общие настройки системы.
  3. Добавьте SMTP-подключение или настройте способ отправки.
  4. Создайте категории подписчиков.
  5. Импортируйте базу подписчиков или подключите форму подписки.
  6. Создайте макросы при необходимости.
  7. Подготовьте шаблон письма.
  8. Создайте запись в расписании рассылок.
  9. Настройте cron на сервере.
  10. Проверьте логи после тестовой отправки.

10. Импорт базы подписчиков

В административной панели предусмотрены отдельные действия для импорта и экспорта подписчиков. Для импорта поддерживаются текстовые и табличные форматы. Перед массовой загрузкой рекомендуется:

  • проверить корректность email-адресов;
  • удалить дубликаты из исходного файла;
  • заранее определить категории, в которые будут загружаться подписчики;
  • проверить кодировку файла.

11. Макросы и персонализация

Раздел Macros предназначен для подстановки переменных в шаблоны писем. Макросы можно использовать для обращения по имени, подстановки email, ссылок подтверждения подписки и других пользовательских данных. Это позволяет создавать более персонализированные письма без ручной правки каждого сообщения.

12. Логи и статистика

В системе присутствуют отдельные сущности для логов, редиректов, процессов и подготовленных отправок. Это означает, что приложение хранит как минимум:

  • состояние отправки писем;
  • отдельные записи по переходам по ссылкам;
  • служебные процессы, связанные с отправкой;
  • отчёты, доступные для скачивания из панели управления.

13. Пользователи и безопасность

  • доступ к панели защищён авторизацией;
  • в системе используется middleware Laravel, включая CSRF-защиту;
  • для подписки и отписки применяются токены;
  • поддерживается разделение прав доступа между пользователями;
  • рекомендуется ограничить доступ к админ-панели по IP или дополнительной HTTP-аутентификацией.

14. Рекомендации по эксплуатации

  • используйте SMTP вместо mail() для лучшей доставляемости;
  • настройте DKIM, SPF и при необходимости DMARC на домене отправителя;
  • запускайте тестовую рассылку перед массовой отправкой;
  • регулярно очищайте неактуальные логи и временные файлы;
  • проверяйте корректность ссылок подписки, отписки и отслеживания;
  • храните резервные копии базы данных перед обновлениями.
