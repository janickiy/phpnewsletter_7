PHP Newsletter — User Manual

PHP Newsletter is a self-hosted email marketing and subscriber management system. It lets you manage mailing lists, build templates, schedule campaigns, track opens and clicks, and keep all data on your own server.

This manual is based on the actual project structure: Laravel 10, admin modules, setup wizard, subscription endpoints, HTML email tracking, cron-based scheduling, user roles, import/export, and reporting.

Who this product is for: website owners, marketing teams, editors, and administrators who need a private email campaign platform without relying on external SaaS providers.

1. System Purpose

PHP Newsletter is suitable for:

  • newsletters and editorial digests;
  • marketing campaigns and promotional mailings;
  • service notifications and website announcements;
  • internal or member-only mailing lists.

It allows you to:

  • manage subscriber lists and categories;
  • create HTML and plain text campaigns;
  • personalize messages with macros;
  • attach files to templates;
  • schedule campaigns for future delivery;
  • collect open and click statistics;
  • run the entire system on your own infrastructure.

2. Quick Start

  1. Install the application and complete the setup wizard.
  2. Log in to the admin panel as the administrator.
  3. Configure the delivery method: SMTP, mail(), or sendmail.
  4. Create subscriber categories.
  5. Add or import subscribers.
  6. Create a template and define macros if needed.
  7. Send a test email.
  8. Create a schedule entry and configure cron.
  9. Review logs and reports after the first live mailing.

4. Installation

  1. Upload or extract the project into the website directory.
  2. Configure your virtual host or domain to point to the application.
  3. Make sure storage and bootstrap/cache are writable.
  4. Open the installer in a browser:
    http://your-domain/install/
  5. Complete the setup wizard steps:
    • system requirement check;
    • permission check;
    • database configuration;
    • administrator creation;
    • installation completion.
What the installer does: it creates .env, writes database credentials, runs migrations and seeders, generates the application key, and creates the first administrator account.

Writable Directories

The application typically needs write access to:

  • storage/app
  • storage/framework/cache
  • storage/framework/sessions
  • storage/framework/views
  • storage/logs
  • bootstrap/cache

5. First Login and Initial Setup

  1. Open /login and sign in.
  2. Go to Settings and fill in the main sender settings:
    • sender email address;
    • sender name;
    • content type;
    • character set;
    • subscription confirmation behavior.
  3. Go to SMTP and create at least one valid SMTP profile if SMTP will be used.
  4. Verify the site URL and unsubscribe links.
  5. Create a test template and perform a test send.

7. How Mailing Works

  1. You create a message template.
  2. You select one or more subscriber categories.
  3. You either launch a campaign manually or schedule it.
  4. The system selects active subscribers who are eligible for the current send.
  5. For each message, the subject, body, macros, unsubscribe links, and mail headers are prepared.
  6. Send results are written into logs.
  7. For HTML messages, opens and clicks can be tracked.
Important: HTML opens are tracked through a 1x1 tracking pixel. If the email client blocks external images, the open may not appear in reports.

8. Subscribers and Categories

Manual Entry

An administrator can create a subscriber manually by providing a name, email address, and one or more categories. Each subscriber receives a token used for subscribe/unsubscribe links.

Import

Supported formats:

  • TXT
  • CSV
  • XLS
  • XLSX
  • ODS

Before import, it is recommended to:

  • remove duplicates;
  • validate email addresses;
  • prepare destination categories in advance;
  • check file encoding.

Export

Subscriber export is available in text and spreadsheet formats. This is useful for backups, migrations, and external analysis.

Subscription Form

The application includes built-in public subscription endpoints:

  • /form — subscription form;
  • /categories — category list for the public form;
  • /add-sub — add a new subscriber;
  • /unsubscribe/{subscriber}/{token} — unsubscribe link;
  • /subscribe/{subscriber}/{token} — subscription confirmation link.

The subscription form can also be embedded into your own website. In the admin panel, open the Subscription Form page to preview the form and copy the ready-made HTML/JavaScript embed code. Place this code on the desired page of your site; it will load available categories, submit subscriber data through the public /add-sub endpoint, and show success or validation messages without requiring visitors to leave your website.

Embedding note: the copied form code uses jQuery and the public API routes listed above. Make sure the PHP Newsletter installation is reachable from the page where the form is placed, and that the selected subscriber categories are active.

9. Templates, Macros, and Attachments

Templates

A template usually contains:

  • message name or subject;
  • email body;
  • delivery priority;
  • optional attachments.

Macros

Macros are placeholders inserted into templates to personalize outgoing mail. Common use cases include:

  • subscriber name;
  • email address;
  • service links;
  • custom user-related values.

Attachments

Files can be attached to a template and are added automatically during delivery.

10. Email Delivery Methods

Mode Recommended Use
SMTP The recommended production option for reliable email delivery.
mail() Best suited only for simple or temporary setups when the server is properly configured.
sendmail Use if a working sendmail binary is available on your infrastructure.

Key Settings to Review

  • sender email address and sender name;
  • content type: HTML or plain text;
  • character set;
  • sendmail path if this mode is used;
  • send limits per run;
  • sleep interval between messages;
  • resend interval for the same subscriber.

11. Open and Click Tracking

Email Opens

For HTML email, the system injects a 1x1 tracking image into the message body. When the image is loaded, the message is marked as opened.

  • available only for HTML emails;
  • depends on external image loading in the recipient's mail client;
  • useful for general analytics, but never perfectly accurate.

Link Clicks

Links can be routed through an internal redirect endpoint. This allows the system to record click statistics before forwarding the user to the final destination.

12. Scheduler and Cron

The project uses these commands:

  • emails:send — processes scheduled delivery;
  • emails:unsent — retries unsent messages.
If cron is not configured: scheduled campaigns will not start automatically, even if they exist in the Schedule module.

13. Roles and Security

The application includes at least two main roles:

  • admin — full access to all modules, including SMTP, settings, and users;
  • moderator — access to categories, subscribers, and macros.

Security Recommendations

  • use a strong administrator password;
  • restrict admin panel access by IP if possible;
  • create database backups before upgrades;
  • do not use production SMTP credentials in development or test environments;
  • review send logs and error logs regularly.

14. Logs and Reports

The system stores:

  • send history;
  • success and failure statuses;
  • HTML email open data;
  • click-through records;
  • downloadable spreadsheet reports.

This helps answer practical questions such as:

  • how many messages were sent successfully;
  • how many failed and why;
  • how many were opened;
  • which links generated the most clicks.

15. Deliverability Recommendations

  • prefer SMTP over mail() for production campaigns;
  • configure SPF, DKIM, and ideally DMARC for your sender domain;
  • avoid sudden high-volume mailing without warming up your domain and IP;
  • make sure unsubscribe links are valid and reachable;
  • keep attachments and HTML complexity under control;
  • send test campaigns to several mailbox providers before a large mailing.

16. Troubleshooting

Emails Are Not Sent

  • verify SMTP host, port, username, and password;
  • make sure your firewall allows outbound SMTP connections;
  • check send logs and application logs;
  • confirm that cron is active.

Subscriptions Do Not Arrive

  • check that the public subscription form is available and /add-sub works;
  • make sure the email address is not already in the database;
  • verify whether confirmation is enabled and confirmation emails are being delivered.

Open Tracking Does Not Work

  • make sure the campaign is sent as HTML;
  • remember that many mail clients block external images;
  • confirm that the application URL is correct and the tracking pixel is publicly reachable.

Scheduled Campaigns Do Not Start

  • check your cron configuration and php artisan schedule:run;
  • make sure the schedule entry falls into the correct time window;
  • verify the server time zone.

17. Project Structure

  • app/ — controllers, models, services, DTOs, middleware, helper classes, and console commands;
  • routes/ — web, api, and console routes;
  • resources/views/ — admin templates, public subscription pages, and installer screens;
  • database/migrations/ — database schema definitions;
  • storage/ — logs, cache, temporary files, and stored attachments;
  • public/ — publicly accessible assets and entry point.
Summary: PHP Newsletter is a complete self-hosted mailing solution with subscriber management, segmentation, templates, analytics, and scheduling. For stable production use, pay special attention to SMTP configuration, cron, writable directories, and domain DNS records.
